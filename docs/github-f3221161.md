# Github

### **GitHub - Sistema de Repositórios e Colaboração**

O GitHub é a plataforma utilizada para armazenar nossos repositórios de código. Ele facilita o trabalho colaborativo e o versionamento de código, permitindo o acompanhamento e gerenciamento de alterações ao longo do tempo.

Atualmente, temos uma organização no GitHub, que pode ser acessada [aqui](#link).


---

### **Gerenciamento de Usuários**

#### **Adicionar Usuários**


1. Acesse a aba **People** na tela principal.

   ![image.png](/api/attachments.redirect?id=e638879e-440c-48ce-bfa1-9fb770ea506a)
2. No canto superior direito, clique em **Invite Member**.

   ![image.png](/api/attachments.redirect?id=1c758ee8-8787-4059-8f65-9cb1279790ce)
3. Adicione o email ou nome de usuário do membro a ser convidado.


---

#### **Remover Usuários**


1. Acesse a aba **People**.

   ![Captura de tela 2024-09-10 141716.png](/api/attachments.redirect?id=fb4bbac9-d624-4ba1-9950-228fd827956b)
2. Escolha o membro desejado e clique nos 3 pontos ao lado do nome.

   ![image.png](/api/attachments.redirect?id=6e8e1280-0fb6-4867-b68d-65c15d24bdd5)
3. Clique em **Remove from Organization**.


---

#### **Alterar Permissão de Owner**


1. Acesse a aba **People**.

   ![Captura de tela 2024-09-10 141716.png](/api/attachments.redirect?id=fb4bbac9-d624-4ba1-9950-228fd827956b)
2. Selecione o membro desejado e clique nos 3 pontos.

   ![image.png](/api/attachments.redirect?id=6e8e1280-0fb6-4867-b68d-65c15d24bdd5)
3. Escolha **Change Role** e selecione a opção **Owner**.

   ![image.png](/api/attachments.redirect?id=66d0d4a7-b164-4afe-90a4-25338d8cff7c)

> **Importante**: A permissão de Owner concede acesso total à organização, incluindo ações críticas. Esta permissão deve ser concedida apenas quando houver uma necessidade clara.


---

#### **Gerenciar Permissões em Repositórios**


1. Acesse a aba **People**.

   ![Captura de tela 2024-09-10 141716.png](/api/attachments.redirect?id=fb4bbac9-d624-4ba1-9950-228fd827956b)
2. Selecione o membro desejado e clique nos 3 pontos.

   ![](/api/attachments.redirect?id=e4f13f37-b8ee-4f6a-b929-56dc7a2fcd43)
3. Clique em **Manage**.
4. Nessa tela, será possível gerenciar os acessos do usuário em cada repositório da organização.

   ![](/api/attachments.redirect?id=0235664b-db37-4cd6-8a74-1c89bdd609a1)


---

#### **Criar Times**


1. Acesse a aba **Teams**.

   ![image.png](/api/attachments.redirect?id=d94eddf2-ff00-4b7d-b2ba-273b37497a33)
2. Clique em **New Team**.

   ![](/api/attachments.redirect?id=81d00abc-f628-4824-bc79-35b515298948)

   \
3. Preencha as informações básicas e clique em **Create**.

   ![image.png](/api/attachments.redirect?id=4aae4ecf-995f-4685-9690-e13a507685fd)

### **Custos do GitHub 💸**

Nosso plano atual no GitHub tem o custo de **$4 por usuário**. Atualmente, temos **35 usuários ativos**, totalizando um custo mensal de **$$144.0**.


---

### **GitHub Actions**

Utilizamos o **GitHub Actions** para automatizar algumas tarefas no processo de desenvolvimento e entrega de software.

* O plano **GitHub Team** nos oferece **3000 minutos gratuitos** para rodar automações.
* Após ultrapassar esse limite, há uma cobrança por minuto de execução, que pode ser consultada [aqui](https://docs.github.com/en/billing/concepts/product-billing/github-actions).
* Temos um limite de **$50** para o custo com GitHub Actions.
