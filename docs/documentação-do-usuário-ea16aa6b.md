## 2. Visão Geral da Interface

Ao acessar o sistema, nota-se que a tela principal é dividida em áreas bem definidas.

### **2.1. Painel de Filtros (Parte Superior)**

O painel de filtros é utilizado para restringir os resultados no mapa e contém os seguintes pontos:

 ![Painel de filtros](/api/attachments.redirect?id=e37ae963-1ccd-4f49-861c-ac7138c5fb89 " =855x117.5")

* **Bairro:** lista suspensa com busca para selecionar um bairro específico.

  \
* **Cidade:** lista suspensa com busca para selecionar uma cidade específica (por padrão, aparece "Florianópolis, SC").

  \
* **Canal:** lista suspensa com busca para selecionar um canal específico.

  \
* **Motivo de Perda:** filtra terrenos de acordo com a razão pela qual não foram adquiridos.

  \
* **Etapa:** lista suspensa que permite filtrar os terrenos que estão em determinada etapa do processo.

  \
* **Data de Fechamento:** permite filtrar por um intervalo de datas.

  \
* **Data de Inserção:** permite filtrar por um intervalo de datas.

  \
* **Parceiro:** lista suspensa com busca para selecionar um parceiro.

  \
* **Etapa de proposta:** checkbox que ao ser ativado mostra no mapa os terrenos em aberto no funil de proposta.

  \
* **Botão "Aplicar Filtros":** atualiza o mapa com base nos filtros definidos pelo usuário.

  \
* **Botão "Limpar Filtros":** remove todos os filtros e retorna à visualização padrão.

## **3. Funcionalidades**

### **3.1 Navegação no Mapa**

* A navegação pelo mapa pode ser realizada utilizando os controles de zoom (__[item 2.2](/doc/documentacao-do-usuario-GkOg4UHnUn)__) ou a rolagem do mouse para aproximar ou afastar a visualização. 

  \
* Para movimentar o mapa, basta clicar e arrastar a área de visualização.

  \
* É possível clicar em um marcador ou nos polígonos (microrregiões, regiões de interesse e zoneamento) para abrir uma janela com informações detalhadas sobre aquele terreno ou área.

 ![](/api/attachments.redirect?id=350dbd45-bb85-442f-998f-81a990f326f2 " =570x238")

### **3.2 Aplicação de Filtros**

* Defina critérios nos campos do painel de filtros (__[item 2.1](https://outline.seazone.com.br/doc/documentacao-do-usuario-GkOg4UHnUn#h-21-painel-de-filtros-parte-superior)__)
* Clique em "Aplicar Filtros" para atualizar o mapa e a visualização será correspondente aos filtros definidos
* É possível combinar vários filtros ao mesmo tempo.

O filtro "Data de Fechamento" atua de forma independente dos filtros de cidade ou bairro. Isso significa que, mesmo que você tenha selecionado uma cidade ou bairro específicos, a filtragem pela data vai considerar todas as regiões disponíveis no banco de dados, sem se limitar ao local escolhido. 


### 3.3 Botão "Mostrar/Ocultar IDs"

O botão de "Mostrar/Ocultar IDs" fica indisponível (cor cinza) quando o zoom está baixo e/ou quando a camada de terrenos está inativa. Para que ele fique disponível (cor azul), é necessário ativar a camada de terrenos e aproximar o zoom.

 ![Aparência do botão quando está disponível](/api/attachments.redirect?id=36bad13b-1495-4efa-a4da-c6697204af62 " =142.5x30.5")

### 3.4 Botão "Mostrar/Ocultar nomes"

O botão de "Mostrar/Ocultar nomes" fica indisponível (cor cinza) quando o zoom está baixo e/ou quando a camada de empreendimentos concorrentes está inativa. Para que ele fique disponível (cor azul), é necessário ativar a camada de empreendimentos concorrentes e aproximar o zoom.

### **3.4 Busca por IDs**

Digite o código do terreno no campo "Buscar por ID", selecione a opção da lista ou pressione Enter. O mapa será centralizado com um zoom no terreno correspondente.

### **3.5 Busca por Coordenadas**

Para buscar as coordenadas, insira a latitude e a longitude no formato decimal, utilizando o ponto (.) como separador decimal (ex.: -27.59, -48.55). Ao concluir a digitação, um marcador temporário será automaticamente criado no local indicado.

### **3.6 Gerenciamento de Camadas**

É possível marcar e desmarcar as opções "Terrenos", "Regiões de Interesse", "Microrregiões" e "Zoneamentos" para ajustar as informações visíveis no mapa. Várias camadas podem ser ativadas ao mesmo tempo.

### **3.7 Busca por endereços e lugares**

Há um espaço no mapa destinado a busca por endereços e lugares. Basta digitar o nome do que você deseja buscar que isso será processado e aparecerá no mapa o local buscado com um marcador vermelho e o zoom no local procurado. Abaixo segue um exemplo:

 ![](/api/attachments.redirect?id=d2452603-3c96-4f21-8f2e-d3c2654b396b " =921x317.5")

### 3.8 URL Dinâmica

O sistema permite que um terreno do mapa seja compartilhado por meio de uma URL dinâmica. Ao clicar em um terreno específico, ou seja, ao abrir sua *InfoWindow*, a URL da barra de endereços é automaticamente atualizada com o identificador do imóvel. Isso possibilita que o mesmo ponto seja acessado diretamente por outro usuário ao compartilhar o link. Segue o passo a passo abaixo:

 ![Ao clicar no terreno, a URL é atualizada automaticamente com a referência desse terreno.](/api/attachments.redirect?id=22bbafe5-1b9d-44d8-b266-3302f4ad5111 " =1778x986")

 ![Ao copiar a URL e mandar para outro usuário, o mapa é centralizado exatamente no ponto em que se encontra o terreno](/api/attachments.redirect?id=5864f7ee-792c-4eaa-a615-cdce1bf24d54 " =1771x992")

## **4. Processo de Atualização dos Dados**

### **4.1 Atualização dos Marcadores**

Os dados dos Marcadores (representando a camada Terrenos) são atualizados automaticamente a partir da  mesma fonte de dados (Pipefy) que atualiza o __[BD de Terrenos](https://docs.google.com/spreadsheets/d/1U7E3wCKaGpMOaMlfKdnR-0L1hyBIjOTlb93f_kg6GIM/edit?gid=220134111#gid=220134111)__. Não é  necessário realizar nenhum processo manual para que novos terrenos apareçam ou que os terrenos alterados sejam atualizados no mapa, pois o sistema realiza essa sincronização diversas vezes ao dia, seguindo os horários definidos na Nekt (atualmente às 04:00, 08:00, 09:00, 14:00 e 19:00)

 ![(Horários de atualização da Nekt)](https://lh7-rt.googleusercontent.com/docsz/AD_4nXdp2rjMWodFvciaKQ5iGAQn_yfy1vi0BLEPNV_tlyuz_E8OtLKTpAqwG-mE547hTHnbTLxS1WNWMY6hK0Y2Q90utYE-FElrlBZ3LK_BX2sUW0mncz9lxFQQpUto5HsnyCM4_DIkUA?key=fIjdmwcipur5J1zZUnNrbg " =216x85")

### **4.2 Atualização de Empreendimentos Concorrentes**

* Diariamente nos horários 01h, 05h, 06h, 11h e 16h, é executada uma consulta programada na Google Cloud Platform (GCP), que é responsável por atualizar a tabela fonte de dados para os dados de empreendimentos concorrentes. 

### **4.3 Atualização dos Polígonos**

As camadas de polígonos (Microrregiões, Zoneamento e Polígonos de Interesse) são atualizadas manualmente por meio do envio de arquivos no formato .kml para o link de atualização indicado nesse [link](https://script.google.com/a/macros/seazone.com.br/s/AKfycbxoXw7bt3xJOLztPD98Kvtf23vGBmqXebFBn0b_VFyilzsRAVVqjbm7vNaAcs3LEXdx/exec).

Escolha o arquivo atualizado, clique em "Enviar e Processar" e aguarde uma mensagem de confirmação do carregamento. ![Interface para enviar os arquivos .kml](https://lh7-rt.googleusercontent.com/docsz/AD_4nXc0ABVCgd9c5SLll6FOh1rmLxTYu6Ep-HkfDzlYPi9ngZmp9oefUUnK3njy3kXgxPhye4f4LuXS2-2YRjrcUtUkDBV_Do7X6oes_iCWDE8FEHPzJ7DvjNBIu3CYVdQzICptmHTknQ?key=fIjdmwcipur5J1zZUnNrbg " =505x272")

\nPara garantir que a atualização funcione, é necessário seguir o padrão de nomenclatura estabelecido, dessa forma, o sistema reconhece corretamente cada camada:

* Microrregiões: `Microrregioes.kml`
* Zoneamentos : `Zoneamentos.kml`
* Polígonos de Interesse: `Poligonos de Interesse.kml`
