# Quando meu pedido vai ser feito?

Use o checklist abaixo para definir a prioridade do seu pedido de suporte:


1. É um problema ou é uma funcionalidade nova?

   
   1. Se é um problema, pule para o passo 2
   2. Se é uma funcionalidade nova, pule para o passo 3
2. Problema - algo que já está implementado no site, mas não funciona como deveria

| Prioridade | Descrição | Quando será feito |
|----|----|----|
| P0 | Impede o fluxo de reservas | Imediatamente |
| P1 | Confunde ou atrapalha o usuário, mas não impede o fluxo de reservas | Assim que o próximo desenvolvedor terminar o que está fazendo (4 dias no máximo) |
| P2 | É um problema cosmético, mas que não confunde o usuário e nem impede reservas | Na próxima sprint (1 mês) |

   
   1. Funcionalidade nova - algo que precisaria ser implementado

| Prioridade | Descrição | Quando será feito |
|----|----|----|
| P0 | Mudanças de negócio radicais que bloqueiam processos dentro do seu time ou acarretariam em prejuízo para a empresa | Imediatamente |
| P1 | Teria impacto significativo comprovado no negócio da empresa | Na próxima sprint |
| P2 | Sugestões de melhoria | Será priorizado ao longo do ano de acordo com as prioridades da empresa de acordo com estimativas de impacto x esforço |
