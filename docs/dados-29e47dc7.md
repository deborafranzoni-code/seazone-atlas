# Dados

## Documentação Sob Responsabilidade

O PM de Dados é o responsável primário pela criação, organização e manutenção das documentações que descrevem o "Porquê" e o "O Quê" dos nossos produtos de dados. Exemplos incluem:

* **Documentação de Discovery:** Resumos e achados das fases iniciais de exploração de problemas e oportunidades, incluindo necessidades de stakeholders e definições de alto nível.
* **Roadmaps de Produtos de Dados:** [Planilha Rodmap Produtos Data](https://docs.google.com/spreadsheets/d/1CbCF6t9_NRB6tJRqHKF43vDoYHxCfXFrQOxUf2xlZi4/edit?gid=828962941#gid=828962941)
* **Epics e Descrições de Alto Nível:** Detalhes sobre os objetivos, escopo e valor esperado das grandes iniciativas. [PDA Board](https://seazone.atlassian.net/jira/software/c/projects/PDA/boards/28)
* **KPI Data Products**
* **Definições de Concluído (DoD):** Critérios para considerar uma demanda como finalizada.
* **Análises de Impacto e Casos de Uso:** Justificativas e benefícios esperados dos projetos.
* **Documentação de Produtos (Visão de Negócio/Usuário):** Documentos que explicam a funcionalidade e o valor dos produtos para os usuários (não a documentação técnica de implementação).
