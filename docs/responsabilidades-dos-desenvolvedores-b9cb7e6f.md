# Responsabilidades dos Desenvolvedores

| **Cargo** | Compreensão da Task | Qualidade | Monitoramento | Code Review | Comunicação | Proatividade |    |
|----|----|----|----|----|----|----|----|
| Júnior | É esperado que consiga desenvolver a task, dado que todas as informações tanto de tecnologia quanto de produto foram dadas | É esperado que consiga definir casos de teste para a sua tarefa | Monitoramento em produção com ajuda do líder técnico | É esperado que faça perguntas na PR para entender porque as coisas foram feitas de um jeito ou de outro. | É esperado que peça ajuda sempre que travar em alguma task e comunique possíveis atrasos frente ao prazo esperado. É esperado que estime o tempo que uma task levaria com ajuda. | É esperado que consiga utilizar o nosso site e encontrar bugs |    |
| Pleno | É esperado que consiga desenvolver a task dado que todas as informações de negócio foram dadas, mas que saiba como desenvolver toda a parte de tecnologia sem ajuda | É esperado que defina seus próprios casos de teste e encontre seus próprios bugs em produção | Monitoramento em produção de todas as suas features novas e em experimentação | É esperado que defina casos de teste para testar o código do par e valide o código para todos esses cenários. | É esperado que consiga estimar o tempo que levaria fazendo a task. | É esperado que além de bugs, consiga trazer sugestões de performance e segurança |    |
| Sênior | É esperado que além de desenvolver a task, investigue como esse código operaria em situações futuras como aumento de usuários, mudanças de regras de negócio, etc | É esperado que defina seus casos de teste e encontre seus próprios bugs antes da task ir para produção | Monitoramento ativo em produção de todo o escopo do time. É esperado que seja o primeiro a detectar e comunicar problemas em produção. | É esperado que que sugira aos pares como melhorar o código para operar em *situações* futuras como aumento de usuários, mudanças de regras de negócio, etc | É esperado que que consiga estimar o tempo que levaria fazendo a task e cuidar de todo o processo para que essa task seja feita e entregue no prazo. | É esperado que além de bugs, performance e segurança, consiga avaliar novas tecnologias para agregar ao produto |    |
