# 🔑 Permissões

Antes de começar a botar a mão na massa, é importante garantir que você tenha acesso às principais ferramentas que usamos no dia a dia. Algumas delas são essenciais para começarmos o setup do ambiente, então, é crucial já ter essas permissões para que o processo de onboarding ocorra sem entraves.

### Google Cloud Platform (GCP)

**➡️** Nossa plataforma de nuvem secundária, utilizada, no contexto de Hosting, para suportar o processo de SSO do Google e alguns outros processos paralelos na Wallet.


:::success
Para ter acesso ao ambiente da GCP, abra um chamado no `[#suporte-governanca](https://seazone-fund.slack.com/archives/C07EAUZQQ7Q)` solicitando a devida permissão.

:::


---

### Pritunl

**➡️** Nossa Virtual Private Network (VPN), utilizada principalmente para acessar de forma segura aos ambientes privados da Seazone, como bancos de dados e outras ferramentas.


:::success
Para ter acesso à VPN, abra um chamado no `[#suporte-governanca](https://seazone-fund.slack.com/archives/C07EAUZQQ7Q)` solicitando a liberação da ferramenta.

:::


---

### PostHog

**➡️** Nossa ferramenta de *web analytics*, usada para acompanhar eventos e métricas de uso das aplicações, mas que também tem um papel crucial para a validação de uso dos usuários — pelos **[Session Replays](https://posthog.com/session-replay)** — e para liberação condicional de features em produção — com **[Feature Flags](https://posthog.com/docs/feature-flags)**.


:::success
Para conseguir acesso ao PostHog, basta solicitar o convite da ferramenta a qualquer um dos @[Membros](mention://fe740c0c-6671-4379-b1c2-84ade6169ee9/document/7eabd241-9db2-487b-ad0f-2243258362e8) do time.

:::


---

### Pipefy

**➡️** Plataforma usada pelo time para receber e acompanhar **tickets operacionais**, como reports de bugs e outras tarefas relacionadas aos nossos produtos. É por aqui que muitas demandas do dia a dia chegam, então vale a pena ter sempre por perto.


:::success
**P**ara conseguir acesso ao Pipefy, solicite acesso à @[@Isabel Karina Villanes Rojas](mention://49bb0802-cf6c-4fcc-963f-da9fed524444/user/20539656-5f0a-4b1a-941b-02ebb6fdc5e7).

:::


---

### GitHub (seazone-tech)

**➡️** Organização do GitHub onde todo o código das nossas aplicações vive. Aqui ficam os repositórios da Seazone, incluindo os projetos da Wallet e do Sapron.


:::success
Para ter acesso ao GitHub da Seazone, abra um chamado no `[#suporte-governanca](https://seazone-fund.slack.com/archives/C07EAUZQQ7Q)` solicitando a devida permissão.

:::


---

### Amazon Web Services (AWS)

**➡️** Nossa principal plataforma de cloud onde as aplicações são hospedadas. No dia-a-dia, usamos principalmente o S3 para armazenamento de objetos e o Parameter Store do SSM para gerenciar variáveis de ambiente.


:::success
Para ter acesso ao ambiente da AWS, abra um chamado no `[#suporte-governanca](https://seazone-fund.slack.com/archives/C07EAUZQQ7Q)` solicitando a devida permissão.

:::


---

### Sapron

**➡️** O Sapron é um dos sistemas que o time de Hosting mantém. Ele concentra diversas ferramentas de apoio ao time de backoffice e franquias, mas, no nosso contexto, é utilizado bastante para validar algumas coisas em produção e acessar as ferramentas de perspectiva (forma de entrar na conta de um usuário, em produção, com permissões limitadas). 


:::success
Para ter acesso ao Sapron, abra um chamado no `[#suporte-governanca](https://seazone-fund.slack.com/archives/C07EAUZQQ7Q)` solicitando a devida permissão.

:::


---

### Bancos de Dados (stg e prod)

**➡️** Credenciais de banco de dados de staging (stg) e produção (prod), usadas tanto para o acesso às informações do banco em si, quanto para manipulação dos dados, conforme necessidade.


:::success
Para ter acesso aos bancos, abra um chamado no `[#suporte-governanca](https://seazone-fund.slack.com/archives/C07EAUZQQ7Q)` solicitando a devida permissão.

:::


---

### Outline

**➡️** Nossa principal ferramenta de documentação — tipo essa aqui :D. Algumas features demandarão um discovery prévio ou documentação de uso, então, é importante ter a permissão de escrita aqui, só pra não ficar preso na task.


:::success
Para conseguir o acesso de escrita no Outline, abra um chamado no `[#suporte-governanca](https://seazone-fund.slack.com/archives/C07EAUZQQ7Q)` solicitando a devida permissão.

:::
