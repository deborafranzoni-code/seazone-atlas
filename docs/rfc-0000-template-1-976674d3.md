# :sparkles: Proposta

> **Como o problema será resolvido?**\nDescreva a solução técnica, mudanças previstas e justificativas. Inclua fluxos, exemplos ou riscos.

### 🧱 Arquitetura / Plano de Implementação

* Introduzir Redis como cache compartilhado para as consultas de produtos.
* Cachear os resultados do `/api/products` por 5 minutos usando hash dos parâmetros como chave.
* Invalidar o cache manualmente em atualizações de produtos ou via pub/sub.

### 🖥️ Considerações Backend

* Usar `aioredis` para acesso assíncrono ao Redis.
* Garantir fallback para o banco caso o Redis falhe.
* Incluir métricas de Cache HIT/MISS.

### 🔍 Exemplo

✅ Exemplo bom:

```python
import aioredis

redis = await aioredis.from_url("redis://localhost")

cache_key = f"products:{query_hash}"

cached_data = await redis.get(cache_key) if cached_data: return json.loads(cached_data)

result = fetch_from_db() await redis.set(cache_key, json.dumps(result), ex=300) return result
```

❌ Exemplo ruim:\nAdicionar Redis e ver o que acontece. A gente ajusta depois.
