# Funcionalidade Conciliação OMIE - Sapron Labs

**Conciliação de Pagamentos OMIE - Resumo da Funcionalidade**\n\n  **O que é?**\n\n A Conciliação de Pagamentos OMIE é uma nova ferramenta no Retool (dentro do Sapron) que substitui o processo manual feito via planilhas e macros no Google Planilhas. O objetivo é\n  centralizar todos os registros de conciliação de pagamentos em uma única fonte de dados.\n\n  Importante: Conciliação de Reservas e Conciliação de Pagamentos são processos distintos.\n\n  ---\n  **Como funciona?**\n  O fluxo possui 4 etapas principais:\n\n  1. Upload do Extrato de Pagamento da OTA\n  - O usuário faz upload do CSV de extrato da OTA (Airbnb, Booking, Expedia ou Decolar) pela interface do Retool\n  - O sistema lê o extrato e insere os registros de pagamento nas tabelas internas\n  - Cada reserva pode ter N registros de pagamento, cuja soma deve totalizar o montante da reserva\n\n  2. Busca de Contas a Receber na OMIE\n  - O sistema consulta a API da OMIE para buscar os pagamentos registrados lá\n  - Pagamentos não conciliados são identificados pelo campo observação vazio na resposta da OMIE\n  - Esses pagamentos são armazenados internamente para o processo de match\n\n  3. Match (Conciliação)\n  - O sistema compara os pagamentos da OTA com os pagamentos da OMIE\n  - A busca considera apenas pagamentos dentro de um período específico por OTA:\n\n  ┌─────────┬────────────────────┐\n  │   OTA                │  Período de busca  │\n  ├─────────┼────────────────────┤\n  │ Airbnb              │ 7 dias anteriores  │\n  ├─────────┼────────────────────┤\n  │ Booking           │ 40 dias anteriores │\n  ├─────────┼────────────────────┤\n  │ Expedia            │ 7 dias anteriores  │\n  ├─────────┼────────────────────┤\n  │ Decolar            │ 6 dias anteriores  │\n  └─────────┴────────────────────┘\n\n  - O pagamento é conciliado quando o valor do registro OMIE é encontrado no resultado agregado (agrupado por identificador de pagamento) dos pagamentos de reservas dentro do período\n\n  4. Envio para OMIE\n  - Após a conciliação, o sistema envia automaticamente para a OMIE a informação de que o pagamento foi conciliado, registrando o ID do pagamento da OTA no campo de observação\n  - Se a API da OMIE estiver fora do ar, o sistema retorna um erro e o usuário deve repetir o procedimento depois. Os pagamentos já conciliados internamente ficam marcados como\n  pendentes de envio para a OMIE
