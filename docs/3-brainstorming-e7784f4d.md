# 3. Brainstorming

1. Indicar visualmente que o link do pré check-in foi enviado;
 2. Indicar visualmente que o link do pré check-in foi respondido;
 3. Trazer os dados preenchidos pelo hóspede na rota de completar reservas, campo de editar dados;
 4. Tornar obrigatório o preenchimento de hóspede principal (todos os dados) e acompanhantes (Nome completo, documento e foto de documento) para imóveis que não se caracterizem como hotel ou resort. Para estes casos é obrigatório somente todos os dados do hóspede principal;
 5. Anfitrião deve ter visibilidade na rota de controle, card de check-in, dos dados que foram preenchidos, com possibilidade de edição caso os hóspedes alterem os dados de seus acompanhantes, deve conter ainda a opção de copiar todos os dados do hóspede principal e acompanhantes, com a opção de visualizar o documento;
 6. Para todos os imóveis deve aparecer uma mensagem de proteção de dados;
 7. A opção de configuração de camas deve aparecer somente para imóveis fase 1, com uma mensagem de observação de que está sujeito a disponibilidade;
 8. A opção de solicitação de berço deve ser incluída, apenas para quartos de hotel, com uma mensagem de observação de que está sujeito a disponibilidade;
 9. A inclusão de pets deve ser incluída e aparecer somente para quartos e apartamentos, com uma mensagem de observação de que esta inclusão deve ser aprovada pelo atendimento e inclui um valor extra de cobrança;
10. Informar o hóspede que o anfitrião entrará em contato caso necessite da placa do veículo
