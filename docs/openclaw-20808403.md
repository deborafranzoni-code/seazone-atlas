## 2. Entrar no container

Ao conectar na EC2, execute:

```bash

sudo docker exec -it openclaw-gateway sh
```

## 3. Criar o agente

```bash
# Cria o agente com um workspace isolado

openclaw agents add meu-agente --workspace ~/.openclaw/workspaces/meu-agente
```

Isso cria:

* Workspace: `~/.openclaw/workspaces/meu-agente/` (com [AGENTS.md](http://AGENTS.md), [SOUL.md](http://SOUL.md), [USER.md](http://USER.md), [IDENTITY.md](http://IDENTITY.md))
* State dir: `~/.openclaw/agents/meu-agente/agent/`
* Sessions dir: `~/.openclaw/agents/meu-agente/sessions/`

## 4. Configurar identidade

```bash
# Via flags diretas

openclaw agents set-identity --agent meu-agente --name "Meu Agente" --emoji "🤖"

# Ou editar IDENTITY.md no workspace e carregar

openclaw agents set-identity --workspace ~/.openclaw/workspaces/meu-agente --from-identity
```

## 5. Personalizar o workspace

Editar os arquivos no workspace (`~/.openclaw/workspaces/meu-agente/`):

| Arquivo | Função |
|----|----|
| `AGENTS.md` | Instruções operacionais (system prompt) |
| `SOUL.md` | Persona e tom de voz |
| `USER.md` | Contexto sobre os usuários |
| `IDENTITY.md` | Nome e emoji do agente |
| `TOOLS.md` | Notas sobre ferramentas disponíveis |
| `HEARTBEAT.md` | Checklist para operações periódicas |

## 6. Vincular a um canal Slack

```bash
# Bind ao canal Slack (account = nome da conta no openclaw.json)
openclaw agents bind --agent meu-agente --bind slack:minha-conta-slack
```

**Nota:** Para funcionar no Slack, é preciso ter um **Slack App separado** (com `botToken` e `appToken` próprios) configurado em `openclaw.json` → `channels.slack.accounts`.

## 7. Verificar

```bash
# Listar agentes

openclaw agents list

# Listar com bindings

openclaw agents list --bindings

# Ver bindings específicos

openclaw agents bindings --agent meu-agente
```

## 8. Testar o agente

```bash
# Enviar uma mensagem direta ao agente

openclaw agent --agent meu-agente --message "Olá, quem é você?"

# Enviar e entregar a resposta no Slack

openclaw agent --agent meu-agente --message "Status report" --deliver --reply-channel slack --reply-to "#meu-canal"
```

## 9. Restart do gateway (para aplicar mudanças no config)

```bash
# De dentro do container

openclaw gateway restart

# Ou de fora (na EC2)
docker compose restart openclaw
```

## Configuração manual (alternativa ao CLI)

Se preferir editar o `openclaw.json` diretamente em `/opt/openclaw/config/openclaw.json`:

```json
{
  "agents": {
    "list": [
      {
        "id": "meu-agente",
        "name": "Meu Agente",
        "workspace": "~/.openclaw/workspaces/meu-agente",
        "identity": {
          "name": "Meu Agente",
          "emoji": "🤖"
        }
      }
    ]
  },
  "bindings": [
    {
      "agentId": "meu-agente",
      "match": {
        "channel": "slack",
        "accountId": "minha-conta-slack"
      }
    }
  ]
}
```

## Referência rápida de comandos

| Comando | Descrição |
|----|----|
| `openclaw agents list` | Listar agentes |
| `openclaw agents list --bindings` | Listar agentes com bindings |
| `openclaw agents add <nome> --workspace <path>` | Criar agente |
| `openclaw agents delete <nome>` | Deletar agente |
| `openclaw agents bindings` | Listar todos os bindings |
| `openclaw agents bind --agent <id> --bind <canal:conta>` | Vincular agente a canal |
| `openclaw agents unbind --agent <id> --bind <canal:conta>` | Desvincular |
| `openclaw agents unbind --agent <id> --all` | Remover todos os bindings |
| `openclaw agents set-identity --agent <id> --name/--emoji/--avatar` | Definir identidade |
| `openclaw agent --agent <id> --message "texto"` | Executar turno do agente |
