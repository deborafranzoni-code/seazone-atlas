# Padrão de projeto

* Nomes de variáveis e funções usando convenção de nomenclatura camelCase. Ex: `const dailyPrice = 100`
* Seguir orientações do clean code
* Nomes de imagens devem conter o prefixo img. Ex: `img_casa_em_jurere.png`
* Nomes de ícones devem conter o prefixo icon. Ex: `icon_seazone.png`
* Nomes de logo devem conter o prefixo logo. Ex: `logo_seazone.png`
* Nomenclatura de variáveis de cores do sistema deve seguir nomes estabelecidos no [Design system](https://www.figma.com/file/jUejh7eezhs5pJbX7UJGJi/Design-System-Seazone?node-id=0%3A1&t=yvCTFZUyUZhd58Vp-0)
* Componentes e páginas devem ser responsivos.
* Todo Pull Request (PR) passa por Code Review (CR) da equipe de front
* Usar corretamente as tags do html - **[HTML ssemântico](https://www.w3schools.com/html/html5_semantic_elements.asp)**
* Cada página pode ter apenas 1 H1
