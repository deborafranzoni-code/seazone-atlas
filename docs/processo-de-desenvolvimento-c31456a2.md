# Processo de upstream 🆙

Este processo é utilizado para validar uma ideia de OKR para o time. O objetivo é filtrar as ideias até encontrar aquela que esteja alinhada tanto com os objetivos da área quanto com os objetivos da empresa.

## 1️⃣ New ideas

Nesta etapa do fluxo, não existem muitas regras pré-definidas. Funciona como um backlog onde qualquer membro da equipe pode adicionar novas ideias. Se você tem uma sugestão que pode ser desenvolvida pelo time como um OKR (Objective and Key Result), basta registrá-la aqui como uma nova ideia.

## 2️⃣ Formulário de validação

Este é o primeiro filtro do nosso processo. Para que uma ideia avance da fase de "novas ideias" para a próxima etapa, é necessário que ela atenda a determinados requisitos e forneça informações específicas. Esses critérios são fundamentais, pois serão utilizados para avaliação e desenvolvimento nas etapas subsequentes.

Aqui utilizamos a matriz BASICO para validar se a ideia está apta ou não para passar para a próxima fase, o objetivo é validar se ela consegue responder todas as perguntas da matriz de forma clara e objetiva as perguntas são :

| Ponto analisado | Pergunta a ser respondida |
|----|----|
| Benefícios | Quais benefícios essa ideia gera para  organização ? |
| Abrangência de resultados | Como e quanto essa ideia impacta a organização e o fluxo de valor de desenvolvimento ? |
| Satisfação do cliente interno | Como e quanto essa ideia impacta a satisfação do nosso cliente interno ? |
| Investimento requerido | Quanto dinheiro e tempo investidos precisamos pra implementar essa ideia ? |
| Cliente externo satisfeito | Como e quanto essa ideia impacta a satisfação do nosso cliente externo? |
| Operacionalização | Qual é a complexidade de implementar essa ideia ? |

## 3️⃣ Validated

Nesta fase são consideradas as atividades que foram aprovadas no formulário de validação. Com base nas informações coletadas, essas atividades são classificadas por prioridade, utilizando uma escala que varia de U0 a U2.

### Status dessa etapa 👇

U0 : Maior prioridade

U1 : Prioridade média

U2 : Menor prioridade

## 4️⃣ Analysis

Nesta etapa realizamos uma análise que é dividida em duas partes fundamentais:


1. **Análise Financeira:**
   * Avaliamos se a ideia proposta é financeiramente viável, ou seja, se ela contribui para a economia ou gera custos adicionais. A análise financeira busca identificar a sustentabilidade econômica da ideia a longo prazo.
2. **Análise Técnica:**
   * Verificamos se a ideia é tecnicamente viável e se faz sentido do ponto de vista operacional. Essa análise inclui a avaliação da capacidade de execução com os recursos e tecnologias disponíveis, bem como a identificação de possíveis desafios técnicos que precisam ser superados.

<aside> 💡 É importante destacar que algumas ideias apresentam um impacto econômico significativo apenas a longo prazo. Portanto, é essencial realizar uma análise detalhada para compreender plenamente os benefícios financeiros potenciais ao longo do tempo.

</aside>

### Status dessa etapa 👇

Technical : Análise de viabilidade técnica

Costs: Análise de viabilidade financeira

## 5️⃣ Ready for Development

Quando uma ideia passa por todas as etapas anteriores, ela chega a esta fase, onde, do ponto de vista da equipe, a ideia é considerada viável e valiosa para desenvolvimento. No entanto, ainda é necessário alinhar a proposta com os objetivos estratégicos da organização.

## 6️⃣ Select for development

Esta é a última etapa do fluxo de upstream. Quando uma ideia chega a esta fase, significa que ela foi validada tanto pela equipe quanto pela organização, atendendo a todos os critérios de viabilidade e alinhamento estratégico. Com a validação completa, a ideia está entregue ao fluxo de delivery, que será responsável por seu desenvolvimento e implementação.

# Processo de delivery  📨

<aside> 📨 O processo de delivery é o processo que acontece após a etapa de **upstream** ser finalizada e consiste em dividir as ideias aprovadas em epicos, estórias e tasks para o time desenvolver ao longo do trimestre

</aside>


---

## 1️⃣ Grooming

Esta reunião ocorre no início de cada trimestre e tem como objetivo principal transformar os OKRs (Objetivos e Resultados-Chave) que passaram pela fase de upstream em atividades menores e mais gerenciáveis para a equipe. Esta reunião é essencial para assegurar que todos os membros da equipe estejam alinhados e preparados para executar suas tarefas de forma eficiente, contribuindo para o sucesso dos projetos.

Para mais informações sobre nossas cerimônias acesse aqui 👇

[Rituais do time de governança](/doc/rituais-do-time-de-governanca-WvvFGd36ZH)

## 2️⃣ Todo

Este é o primeiro status de uma tarefa de delivery ao sair do backlog. Nesta fase, ocorre uma reunião semanal, conhecida como "Reabastecimento", para decidir quais atividades serão movidas para a lista de "TODO" durante o período. Este processo garante que a equipe tenha um fluxo contínuo de trabalho, com atividades claramente definidas e prontas para execução

<aside> ⚠️  É importante esclarecer que nem todas as tarefas encaminhadas para a fase de entrega (delivery) são derivadas dos Objetivos e Resultados-Chave (OKRs) da organização, ou têm a mesma prioridade de execução imediata. Por isso, é fundamental que exista um filtro eficaz para garantir que as tarefas selecionadas realmente façam sentido no contexto atual.

</aside>

## 3️⃣ WIP

Esta é a etapa mais simples do processo e indica que a atividade está em andamento. É importante monitorar esta fase para garantir a eficiência do fluxo de trabalho. Manter um fluxo de trabalho equilibrado nesta etapa ajuda a garantir que as tarefas sejam concluídas de forma eficaz e dentro dos prazos estabelecidos

### Status dessa etapa 👇

* **IN PROGRESS :** Esse status indica que a tarefa está em desenvolvimento e tudo está correndo como deveria
* BLOCKED :  Significa que há algum travamento no desenvolvimento da tarefa

## 4️⃣Test

Esta etapa serve como uma validação essencial para garantir a qualidade e a completude da tarefa executada. É recomendado que um relatório ou link que demonstre a tarefa realizada seja adicionado nos comentários do card correspondente.

### Status dessa etapa 👇

* FOR REVIEW : Indica que a tarefa está pronta para ser revisada e ou testada
* IN REVIEW :  Indica que a tarefa já está sendo testada e validada

## 5️⃣ Done

Essa é a última etapa do fluxo e indica que a atividade foi finalizada
