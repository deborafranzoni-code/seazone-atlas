# 📖 Dicionário de Hosting

Durante o dia a dia no time de Hosting, você vai se deparar com várias siglas, expressões e termos que usamos constantemente em reuniões, tickets e conversas no Slack. Para evitar confusões e facilitar sua adaptação, montamos este pequeno dicionário.

Abaixo, você encontra uma tabela com os principais termos e seus significados. Sempre que surgir alguma dúvida sobre **"o que é isso que o pessoal está falando?"**, vale dar uma olhada aqui primeiro. E se não tiver, pode perguntar e já aproveita pra adicionar aqui também :)

| :memo: **Termo** | :thinking_face: **O que é?** |
|----|----|
| **CS / Onboarding** | Refere-se ao time de Customer Success, que tem uma subdivisão (Onboarding) que cuida do processo de integração de novos clientes/usuários. |
| **Discovery** | Etapa inicial de levantamento e validação de necessidades, usada para entender o problema antes de desenvolver uma feature. |
| **Fechamento** | O Fechamento é um processo/cerimônia que realiza os processos envolvendo cálculo e a apuração dos resultados do mês anterior em cada imóvel, efetuando a distribuição de valores aos proprietários, anfitriões, parceiros e à própria Seazone. É um processo financeiro, mas que impacta diretamente os produtos de tecnologia. |
| **FF** | **Feature flag** do PostHog, usada para habilitar ou desabilitar funcionalidades específicas sem precisar fazer deploy. |
| **Franquia** | = Host (para o BD) = Anfitrião (para o negócio). |
| **Grid / Grid Financeiro** | Componente existente na Wallet, onde existe toda a sumarização financeira do proprietário. |
| **OTA** | "Onlive Travel Agency", são as empresas de hospedagem que existem e são nossas parceiras: AirBnb, Decolar, Booking.com e etc. |
| **P0 / P1 / P2 …** | Aquilo que a gente teme, mas que paga nossas contas: bugs. Esses termos estão relacionados ao grau de prioridade do bug. Vide: @[Processo de Suporte Wallet](mention://765ae6ad-57c5-4cbd-9e69-5de66dc82ac1/document/f45c2123-f7c5-46d7-8580-d1fc63b1e908) |
| **Perspectiva** | Ferramenta que desenvolvemos no Sapron para ter a visão de um cliente em produção, sem suas credenciais. Atualmente, existe a versão do [proprietário](https://sapron.com.br/painel-perspectiva-do-proprietario) e do [anfitrião](https://sapron.com.br/painel-perspectiva-do-anfitriao). |
| **Prop** | Proprietário |
| **QA** | "Quality Assurance", etapa de testes e validações que garante a qualidade do software antes de ir para produção. |
| **Repasse** | Se refere ao montante repassado ao proprietário após o fechamento de cada mês de operação do seu imóvel. Basicamente é o Pix que pinga na conta do prop. |
| **Stays** | Nossa principal parceira quando o assunto é gestão de canais. É por lá que acontece toda a orquestração de hospedagens nas várias OTAs. Também é usada em vários processos críticos.  |
| **Storypoint** | Unidade de medida usada para estimar o esforço necessário para completar uma tarefa ou user story. |
| **Termo / Termos de Uso** | É uma feature que temos na Wallet que renderiza um popup para o usuário contendo aditivos/novos contratos para o imóvel.   |
| **Thread** | São as threads que são feitas no nosso canal Slack. Normalmente, você vai encontrar elas nesse formato: **\[ Assunto \].**<br>A ideia é que esse assunto seja discutido nas respostas de thread. |
| **UI Review** | Revisão da interface (User Interface) para validar se a entrega está alinhada ao design e experiência esperada. |
