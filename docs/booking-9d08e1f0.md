# Booking

Para a documentação detalhada dos scrapers do **Booking.com**, aqui você encontrará as especificações de cada um, incluindo a funcionalidade de cada scraper, sua frequência de execução e os locais onde os dados extraídos são armazenados. A estrutura está organizada para facilitar a consulta e compreensão da arquitetura geral do pipeline e como os diferentes scrapers se interconectam.

### Informações principais da documentação:

* **Funcionalidade**: Explicações sobre os dados que cada scraper coleta, o propósito de cada um e quais parâmetros podem ser configurados.
* **Frequência de Execução**: Descrição da periodicidade com que cada scraper roda – se é diário, semanal, ou em intervalos específicos, dependendo da necessidade de atualização dos dados.
* **Local de Armazenamento dos Dados**: Indicação dos buckets e diretórios onde os dados são enviados após a coleta, incluindo pipelines de processamento adicional e preparação para o Data Lake.
